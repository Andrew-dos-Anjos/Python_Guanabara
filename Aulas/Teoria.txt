todo Aula16_Tuplas:
Uma tupla é o termo dado para alocação de mais de um valor em uma única variável.
Toda tupla é imutável, isso quer dizer que uma vez definidos os itens que a compõem ela não pode ser alterada.
()Tupla; []Lista; {}Dicionário.

todo Aula17_Listas:
A lista permite que haja alteração de valores em seus elementos, o que a diferencia inicialmente da tupla.
Outro fator que a lista permite é a inclusão de mais itens inclusive em sepaços específicos, ex.:
lista = [x, y, z]
lista.append("w") -> [x, y, z, w]; lista.insert(0,"w") -> [w, x, y, z].
Também é possível apagar itens:
del lista[2] -> [w, x, z]; lista.remove("w") -> [x, z]; lista.pop -> [x].
Para elementos com valores numéricos é possível utilizar os comandos:
var = list(range(4, 11)) -> ordena de 4 a 10; var.sort(reverse=True) -> ordena N°s de trás para frente.

todo Aula19_Dicionários:
Similar às listas, dicionários também utilizam variáveis para implementar valores,
com o diferencial de poder especificar a(s) posições com nomes, ex.:
L: dados = ["Pedro", 19] -> print(dados[0]) ↵ "Pedro";
D: dados = {'nome': "Pedro", 'idade': 19} -> print(dados[idade]) ↵ 19.
Para o acréscimo de novos elementos ou remoção:
dados['gen'] = "M" -> {'nome': "Pedro", 'idade': 19, 'gen': "M"};
del dados['idade'] -> {'nome': "Pedro", 'gen': "M"}

Outra possibilidade é a forma de escrita:
filme = {'titulo': "O rei leão",
         'ano': 1994,
         'produtora': "Disney™"
        }

todo Aula20_Funções:
As funções são como um atalho para simplificar comandos por linhas de código personalizadas.
Escrita -- def variavel():, ex¹.:
def linha():
    print("-"*30)
Assim, toda vez que 'linha()' for escrito tera como valor: 'print("-"*30)'
ex².:
def texto(txt):
    print("-"*30)
    print(txt)
    print("-"*30)
* Toda função deve ter duas linhas separando a def do cod principal *
*                                                                   *
texto('Com este comando essa mensagem substitui a linha do meio no lugar de "txt" ;)')

.Argumentos opcionais (15:50)
Os parâmetros opcionais servem para o caso de um escopo não ser definido:
def exemplo(a, b, c=0):
    '''
    Param a: 1º Argumento
    Param b: 2º Argumento
    Param c: 3º Argumento
    '''
    s = a + b + c
    print(s)

exemplo(1, 2)
...
Para o caso do "c" não ser preenchido como no ex., o argumento/parâmetro opcional definido para que ocorra tudo bem com
a execução do código é 0, exibindo assim o num 3.

.Retorno de resultados (31:40)
Em situações como a de cima, porém, com mais de uma operação de soma, é interessante implementar o recurso return
possibilitando assim a var "s" redirecione o valor para cada operação. Ex.:
def exemplo(a=0, b=0, c=0):
    s = a + b + c
    return s

op1 = exemplo(3, 2, 5)
op2 = exemplo(2, 2)
op3 = exemplo(6)
print(f"Os resultados foram {op1}, {op2} e {op3}.")

.Escopo de variáveis (21:50)
Variáveis criadas no código principal são chamadas de variáveis globais. Significa que o valor definido para essa var é
o mesmo considerado para o código inteiro (podendo ser modificado).
No caso de uma var x ser alterada dentro de uma função ela é considerada como var local, ou seja, só é valida dentro
daquela def. Também podendo ser tranformadas em var globais com a expressão global(x). Ex.:
def teste(y):
    global x
    x = 8
    y += 4
    z = 2
    Print(f"Dentro vale {x}") ↵ 8
    Print(f"Dentro vale {y}") ↵ 9
    Print(f"Dentro vale {z}") ↵ 2

x = 5
teste(x)
Print(f"Fora vale {x}") ↵ 8
